# Цепочка Avito Webhook — на каком этапе обрыв

## Этапы (по порядку)

| # | Этап | Что проверяем | Как проверить |
|---|------|----------------|----------------|
| 1 | **Токен** | Client ID/Secret верные, токен получаем | `GET /api/avito/diagnostic` → step 1 ok |
| 2 | **Аккаунт** | Доступ к своему аккаунту | diagnostic → step 2 ok, user_id, name |
| 3 | **Подписка в Авито** | URL зарегистрирован у Авито | diagnostic → step 3, urls содержит наш URL |
| 4 | **Список чатов** | Метод get_chats доступен | diagnostic → step 4 ok |
| 5 | **Чтение сообщений** | Метод get_messages доступен | diagnostic → step 5 → **здесь 402** |
| 6 | **Проверка URL Авито** | При регистрации Авито дергает наш URL | В crm.log строка "Avito webhook request" после регистрации |
| 7 | **Доставка событий** | Авито шлёт POST при новом сообщении | В crm.log "Avito webhook request" при сообщении в чат |

## Текущий результат диагностики

- **Шаги 1–4**: OK (токен, аккаунт, подписка зарегистрирована, чаты видны).
- **Шаг 5**: **402** — «Перейдите на подписку с API мессенджера, чтобы получить доступ к чатам».

Вывод: доступ к **сообщениям** (get_messages) и, по всей видимости, **доставка webhook** требуют платной подписки на API мессенджера. Регистрация URL (шаг 3) проходит, но события не приходят.

## Как проверить у себя

1. Открой в браузере (с авторизацией в CRM):  
   `https://crm.flowcabinet.ru/api/avito/diagnostic`
2. Посмотри `steps`: на каком шаге `ok: false` или `status_code: 402`.
3. После того как кто-то напишет тебе в Авито — открой `crm.log` на сервере (или попроси админа) и поищи строку `Avito webhook request`. Если такой строки нет — Авито наш webhook не вызывало.

## Лог входящих запросов на webhook

В `crm.log` теперь пишется каждая попытка вызова webhook (в том числе проверка при регистрации):

```
Avito webhook request: host=... x-forwarded-for=...
```

Если после регистрации или после нового сообщения в Авито этой строки нет — запросы до нашего сервера не доходят (либо Авито их не шлёт).
